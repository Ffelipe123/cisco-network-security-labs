# 🔐 Cisco Network Security Labs (Packet Tracer Portfolio)

> ⚠️ This is a hands-on cybersecurity portfolio focused on practical network defense techniques.

---

## 📌 Overview
This repository contains a curated set of network security labs built using Cisco Packet Tracer.  
The goal is to demonstrate practical skills in both network engineering and cybersecurity by simulating real-world enterprise scenarios.

---

## 🎯 Objectives
- Design and implement secure network infrastructures  
- Apply Layer 2 and Layer 3 security best practices  
- Simulate real-world attack scenarios and mitigations  
- Build job-ready networking and security skills  

---

## 🧠 Skills Demonstrated
- VLAN Segmentation & Inter-VLAN Routing  
- Access Control Lists (ACLs)  
- Network Address Translation (NAT/PAT)  
- Secure Remote Access (SSH)  
- Layer 2 Security (Port Security, DHCP Snooping, ARP Inspection)  
- Network Hardening & Defense-in-Depth  

---

## 🧪 Labs Included

| Lab | Title | Key Concepts |
|-----|------|-------------|
| 01 | Secure VLAN Segmentation | VLANs, Trunking, ACLs |
| 02 | Layer 2 Hardening | Port Security, DHCP Snooping, ARP Inspection |
| 03 | Secure Management | SSH, Device Hardening |
| 04 | Traffic Control & NAT | ACLs, NAT/PAT |
| 05 | Secure Enterprise Network | Integrated Security Architecture |

---

## 🗺️ Lab Structure

Each lab follows this structure:

```
LABXX-name/
├── topology.png
├── configs.txt
├── verification.md
└── lab.pkt
```

---

## 🛠 Tools & Technologies
- Cisco Packet Tracer  
- Cisco IOS CLI  
- TCP/IP Networking  
- Switching & Routing Concepts  

---

## 🔍 Verification Methodology
Each lab includes validation steps such as:

- `show` commands (e.g., show vlan brief, show ip route)  
- Connectivity tests (ping, traceroute)  
- Security validation through controlled test scenarios  

---

## 🔐 Security Approach
This project follows a defense-in-depth strategy by combining:

- Network segmentation (VLANs)  
- Access control (ACLs)  
- Device hardening (SSH, port security)  
- Attack mitigation (DHCP Snooping, ARP Inspection)  

---

## 🚀 How to Use
1. Open the `.pkt` file in Cisco Packet Tracer  
2. Review the topology diagram  
3. Analyze the configurations  
4. Execute the verification steps
